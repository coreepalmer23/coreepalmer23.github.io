Pitch Classification Model

I developed a machine learning model to classify pitch types using live in-game TrackMan data. The overall model was split into multiple classification stages, with subsequent stages being more fine-tuned to the specific pitch. The model used different features in each stage, based on which mix of features best differentiates between pitches in each specific category. When the pitch type is unknown, the model defaults to either the group classification or unknown.

Key Results

  • Group Split Accuracy: 96.3%
  • Fastballs Accuracy: 95.4%
  • Offspeeds Accuracy: 86.7%
  • Breaking Balls Accuracy: 90.0%

TrackMan Dashboard Scraper and Plotter

I developed a Python script that scraped live pitch and batted ball metrics from the TrackMan Dashboard. It went into the HTML to collect and record various metrics, such as IVB, HB, and velocity. It takes HB and IVB to plot the movement of pitches. When combined with the pitch classification model, it color codes the pitch types for easier visible distinction between pitches.

Key Results

  • Increased live game tagging efficiency
  • Aided in mid-game decision making by coaching staff
  • Allowed for easier visualization of a pitcher's live in-game fatigue

NBA Player Status Predictor

I built a logistic regression model that determines the status of a player as either "elite" or "average". I performed preliminary exploratory data analysis to determine which metrics were most likely to influence the performance of a player. The player's status is determined by their statistcal profile from the 2023-2024 season.

Key Results

  • Model Accuracy: 89.4%
  • A sensitivity of 0.9417 shows that the model is very good at identifying non-elite players, but the specificity of 0.4000 shows that it struggles with identifying elite players
  • The top predictors of players status is playing time (minutes played, games played) and Win Shares (defensive win shares, offensive win shares, win shares)
