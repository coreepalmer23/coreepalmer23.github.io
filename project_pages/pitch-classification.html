Pitch Classification Model

Multi-stage machine learning pipeline for clasifying pitch types from live TrackMan data

Python LightGBM TrackMan Feature Engineering

Overview

I developed a machine learning model to classify pitch types using live in-game TrackMan data. Rather than treating every pitch equally, the model uses a hierarchical classification structure that progressively narrows the possible outcomes at each stage.

Problem

Live game environments require fast and reliable pitch tagging. Existing methods created potential delays and introduced inconsistencies during games. There was a struggle to balance both accurately selecting the pitch type and the rest of the game action.

Methodology

  • Trained on historical MLB pitch data from the 2024 season (got the data from Statcast).
  • Engineered unique physics features to supplment typical pitch metrics.
  • Built a staged classification pipeline in Google Colab using Python.
  • Added confidence thresholds for uncertain pitch types.
  • Use collected TrackMan pitch metrics that were scraped from the live dashboard.

Future Improvements

Future versions could incorportate pitcher-specific calibration and real-time confidence monitoring. By making the model pitcher-specific, it could increase accuracy, as well as removing the possibility that the model could assign a pitch type that a pitcher doesn't throw (ex: calls a pitch a sinker when they only throw a fastball).
